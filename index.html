О dump_viewer-os2

Это -- форк nokde-experimental ветки данного репозитория.

Данный форк был сделан по просьбе автора проекта -- ratnik, поскольку поддержка исходным проектом IBM OS/2 (eComStation) оказалась слишком обременительна для команды разработчиков. Таким образом я буду пытаться поддерживать данный форк на одном уровне с оригинальным dump_viewer, периодически сливаясь с новейшим кодом, внося необходимые изменения, делая билды для OS/2 и, возможно, добавляя новые фичи. Баги в dump_viewer, присутствующие не только в данном билде для OS/2, но и в других, прошу репортить ratnik. OS/2-specific баги прошу репортить сюда. Далее идёт оригинальный readme от лица ratnik, доступный по ссылке выше. Спасибо.

dump_viewer -- База раздач RuTracker.org

О проекте

Вашему вниманию предлагается база раздач крупнейшего битторрент-трекера в России rutracker. Мы постарались представить все раздачи на указанное в заголовке число. Её содержимое: заголовки раздач, описания (без картинок), а так же magnet-ссылки для скачки без захода на трекер. Так же в раздачу входит программа для комфортного использования этой базы. Подробнее смотрите на скриншот выше. Проект был создан, чтобы облегчить доступ пользователей к раздачам в случае проблем с доступностью сайта трекера. Например, когда выводится сообщение "форум временно отключен". Кроме того, эта раздача пригодится, если трекер будет внесен в список заблокированных сайтов. Не хочется, чтобы был даже мельчайший шанс того, что всё, что мы тут вместе сделали за эти годы, потерялось по прихоти чиновников или из-за поломки сервера, к примеру. Пока жива данная раздача, все раздачи трекера тоже живы. Вероятно, раз в месяц нужно будет обновлять эту раздачу.

Использование

Чтобы использовать программу, нужен файл final.txt или final.txt.gz или final.txt.bz2. Два последних варианта являются сжатыми версиями первого. Где взять файлы, см. links.txt

Запустите dump_viewer.exe, введите фразу для поиска в строчку, расположенную в верхней части программы. Нажмите кнопку "Найти" или клавишу ВВОД на клавиатуре. Количество результатов можно изменять в поле "макс.находок". Чем больше находок запросите, тем дольше будет искать.

Дальше смотрим на табличку. Думаю, значение столбцов не нужно объяснять. Обращаю ваше внимание, что по всем столбцам можно сортировать, а по умолчанию результаты отсортированы по количеству загрузок. Если дважды щелкнуть по любой ячейке, значение в ней выделяется и становится пригодным для копирования (Control-C).

Важно: чтобы просмотреть описание раздачи, щелкните левой кнопкой мыши в любое поле, кроме номера раздачи и хеша. Описание будет отображено снизу. Чтобы подгрузить страницу с раздачей и отобразить её снизу, щелкните по номеру раздачи. Чтобы скопировать URL раздачи, щелкните по её номеру правой кнопкой мыши. Чтобы открыть магнитную ссылку, щелкните по ней левой кнопкой мыши, чтобы скопировать -- правой.

Разбор пунктов меню

  • Файл
    • Выбрать базу: указываете путь к файлу final.txt, final.txt.gz или final.txt.bz2, в котором хранится список раздач.
    • Распаковать базу: выбраннй в предыдущем пункте файл final.txt.gz или final.txt.bz2 распаковывается в final.txt, чтобы поиск по нему ускорился. Выбирать final.txt при помощи "Файл - Выбрать базу" после этого не требуется.
    • Выбрать базу описаний: указываете путь к папке, в которой лежат описания (папки 001, 002 и т.д.). Описание раздачи с номером 1234567 лежит в файле 012/01234.tar.gz/01234567 в кодировке UTF-8.
  • Вид
    • base32-хеши: отображать в колонке "хеш" base32-запись хеша. Без этой галочки отображаются HEX-хеши. Большинству клиентов нет разницы, в каком формате хеш, но есть те, которым нужен только base32 (Vuze, к примеру). HEX-хеш считается более современным.
    • Кодировка cp1251: не выбирайте эту опцию. Она заставит программу считать, что файл final.txt закодирован в cp1251. Опция оставлена для совместимости с предыдущим форматом final.txt.

Настройки программа хранит в файле dump_viewer.ini, расположенном в папке с программой.

Для пользователей Microsoft Windows

Устанавливаем OpenSSL для Windows (отсюда)http://slproweb.com/products/Win32OpenSSL.html[http://slproweb.com/download/Win32OpenSSL_Light-1_0_1e.exe]), при установке выбираем Copy SSL DLL to: The Windows system directory (если выберем Open SSL binaries нужно будет прописать в переменную окружения PATH), запускаем dump_viewer.exe

Для линуксоидов

Есть исходники, и вы всегда можете собрать сами (инструкции в INSTALL).
Для ленивых обладателей 32-битных систем в раздаче есть бинарник dump_viewer. Требует Qt и kdelibs. Ну или всегда можно использовать (z)grep.
Для 64-битных линуксов бинарников нет.

Для маководов

Мне собирать не на чем, собирайте сами. Пишите результат на мыло, включу в инструкцию.
